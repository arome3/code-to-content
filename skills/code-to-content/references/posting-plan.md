# Posting Plan Generation Instructions

Implementation guide for generating multi-day content schedules with ready-to-post content for each day.

---

## When to Use

Generate a posting plan when the user:
- Is launching a project and wants coordinated content
- Wants to maximize reach across platforms over time
- Needs a content calendar for consistent posting
- Wants to repurpose one project into multiple posts

## 7-Day Launch Plan

### Plan Structure

```
DAY 1: The Announcement (Maximum reach)
DAY 2: The Technical Deep-Dive (Credibility)
DAY 3: The Story (Emotional connection)
DAY 4: The Use Cases (Practical value)
DAY 5: The Lessons (Wisdom sharing)
DAY 6: The Behind-the-Scenes (Authenticity)
DAY 7: The Recap + Ask (Engagement)
```

## Platform Timing Guide

**Optimal posting times:**

| Platform | Best Days | Best Times |
|----------|-----------|------------|
| Twitter | Tue-Thu | 9-11am, 1-3pm |
| LinkedIn | Tue-Thu | 8-10am, 12pm |
| Dev.to | Mon-Wed | Morning |
| Hacker News | Weekdays | 8-10am ET |
| Reddit | Mon-Thu | 6-9am, 12pm |

**Spacing within day:**
- Primary platform: Peak hours
- Secondary platforms: 2-3 hours later
- Cross-post with slight variation, not identical

## Cross-Platform Adaptation

When generating the same content for multiple platforms:

**Twitter â†’ LinkedIn:**
- Expand thread into flowing paragraphs

- Include relevant hashtags at end
- Soften casual language slightly

Before delivering a posting plan:

```
[ ] All 7 days have complete, ready-to-post content
[ ] Each day has distinct theme/angle
[ ] Content is platform-appropriate
[ ] No repetition of same points across days
[ ] Engagement hooks on each post
[ ] Links included where relevant
[ ] Timing recommendations included

[ ] Narrative arc across the week
```
